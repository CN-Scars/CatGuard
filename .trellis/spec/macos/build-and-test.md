# Build & Test Setup

> XcodeGen single-source-of-truth, standalone logic tests, swift-format CI.

---

## Convention: `project.yml` is the single source of truth

The `.xcodeproj` is generated by `xcodegen generate` and is **git-ignored**. Never edit
the project in Xcode's GUI and commit it; edit `project.yml` and regenerate.

```bash
xcodegen generate   # required after cloning, and after any project.yml change
```

`.gitignore` must include `CatGuard.xcodeproj/`. Also ignore local signing material
(`*.p12 *.pem *.cer`) — never commit private keys.

---

## Pattern: standalone logic tests (no TEST_HOST)

**Problem**: hosting a test bundle in the hardened-runtime, custom-signed app makes
`xcodebuild test` fail with `different Team IDs` (see
[code-signing-and-tcc.md](./code-signing-and-tcc.md)).

**Solution**: for pure-logic units, compile the source files directly into the test
target instead of `@testable import`-ing the app. No `TEST_HOST`, no injection.

```yaml
CatGuardTests:
  type: bundle.unit-test
  sources:
    - path: CatGuardTests
    - path: CatGuard/HitTestGeometry.swift     # tested source compiled in directly
    - path: CatGuard/LockStateManager.swift
```

**Test target selection**: test only side-effect-free logic. Skip UI / system-API modules
(`EventTapManager`, `AuthenticationManager`, `FloatingWindowController`,
`PermissionManager`) — verify those by manual run. Required coverage:

| Suite | Asserts |
|-------|---------|
| `LockStateManagerTests` | every state transition + every illegal transition is a no-op |
| `HitTestGeometryTests` | Y-flip correctness; menu-bar bounds incl. **negative Y regression** |

## Convention: swift-format is the only lint/format tool

Apple's `swift-format` ships with the Xcode toolchain — no third-party deps (keeps the
project "simple, auditable"). Config lives in `.swift-format` (4-space indent, width 100).

```bash
xcrun swift-format format -i -p -r --configuration .swift-format CatGuard CatGuardTests  # apply
xcrun swift-format lint --strict -r --configuration .swift-format CatGuard CatGuardTests # CI gate
```

> **Gotcha**: an unrecognized rule key (e.g. `NoForceUnwrapping`) emits a config warning;
> only use rule names valid for the installed swift-format version.

---

## Convention: CI on macOS runner, unsigned build

CatGuard needs AppKit/CoreGraphics/LocalAuthentication → CI must run on a `macos-*`
runner. CI uses **ad-hoc/unsigned** build (does not depend on the local
`CatGuard Local Dev` cert):

```yaml
xcodebuild ... CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO build
```

CI pipeline order: `xcodegen generate` → `swift-format lint --strict` → build → test.
Triggers on push and PR to all branches. See `.github/workflows/ci.yml`.
